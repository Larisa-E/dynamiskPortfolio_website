Bogsalg (Bookstore) Database

Tech stack: Microsoft SQL Server, T-SQL, Relational Modeling, SSMS

GitHub repository

Highlights

  • Architecture: Built with Microsoft SQL Server, T-SQL, Relational Modeling, SSMS and structured for readability and future extension.
  • Challenge: Modeled relational data carefully to preserve integrity and support realistic reporting/query scenarios.
  • Learning: Strengthened architecture decisions and debugging discipline while turning complex requirements into maintainable output.

View project

Bogsalg_db is a SQL Server project that models a bookstore domain with customers, genres, books, orders, and order line items. I designed the schema with normalized relationships, PK and FK constraints, and data-quality checks such as non-negative values and valid year and page boundaries. The scripts are intentionally idempotent and safe to re-run, including guarded inserts and task-based SQL files for repeatable exercises in INSERT, SELECT, and UPDATE workflows. The project includes ER documentation and screenshot-based proof for each task, showing not just design but also validated execution and results.
