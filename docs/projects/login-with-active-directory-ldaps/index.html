Login with Active Directory (LDAPS)

Tech stack: PHP, LDAP, LDAPS, Active Directory, Windows Server, XAMPP, Sessions

GitHub repository

Highlights

  • Architecture: Built with PHP, LDAP, LDAPS, Active Directory, Windows Server, XAMPP, Sessions and structured for readability and future extension.
  • Challenge: Handled authentication and identity constraints while keeping session handling secure and predictable.
  • Learning: Strengthened architecture decisions and debugging discipline while turning complex requirements into maintainable output.

View project

This project demonstrates enterprise-style authentication by integrating a PHP login system with Microsoft Active Directory using encrypted LDAPS (port 636). Instead of storing passwords in the app, credentials are verified through LDAP bind against the domain controller, then protected routes are unlocked with session-based access control. I implemented login/logout flow, protected welcome page access, and audit logging that records timestamp, username, IP, and success or failure. A major part of the work was operational setup and troubleshooting certificate trust for self-signed LDAPS in a lab environment, including OpenSSL and PHP trust configuration with reproducible setup documentation.
